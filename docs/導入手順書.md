# 導入手順書

## 目的

AI Agentを標準構成で導入し、利用環境の分散、古い設定の残存、未承認接続を防ぐ。

## 1. 導入前確認

| 確認項目 | 内容 |
|---|---|
| **対象端末** | 業務で利用する会社管理PCであること。 |
| **利用目的** | 個人試行、チーム内利用、部内業務利用のどれかを確認する。 |
| **外部接続** | 未承認MCP、Plugin、個人契約API、未承認SaaSを使わない。 |
| **機密情報** | 機密情報、認証情報、本番データを投入しない前提で始める。 |
| **Bitbucket** | 継続利用するコードの保存先を確認する。 |
| **プログラムランタイム** | Python、Node.js等のランタイムが必要な場合は、会社のソフトウェア導入許可リスト・指定手順に従う。 |
| **ライブラリ導入** | pip/npm等のライブラリは、標準インストーラーで配布される許可済みパッケージリストと指定手順に従う。 |

## 2. 導入手順

| 手順 | 作業 |
|---|---|
| **1** | 標準インストーラーを取得する。 |
| **2** | インストーラーを実行する。 |
| **3** | 標準設定、AGENTS.md / CLAUDE.md、Hook、テンプレート、許可済みパッケージリストが配置されたことを確認する。 |
| **4** | 導入確認コマンドを実行し、Agent本体バージョン、標準設定バージョン、設定ハッシュを確認する。 |
| **5** | サンプルで動作確認する。 |

## 3. 導入後の確認

| 確認項目 | 確認内容 |
|---|---|
| **標準版** | 指定された標準版が入っている。 |
| **設定ハッシュ** | 標準設定が改変されていない。 |
| **外部接続** | 許可されていない接続先が追加されていない。 |
| **Sandbox** | 標準設定により、AI Agentが操作できる作業フォルダ・実行範囲が限定されている。 |
| **ライブラリ導入制御** | `pip install` / `npm install` 等が許可済みパッケージリストを参照するHookまたはラッパー経由で確認される。 |
| **ログ** | 導入状況、エラー、拒否イベントを確認できる。 |

## 4. セルフ・セキュリティ・チェック

ローカルリポジトリに仕組みができた場合は、チーム共有・Bitbucketへのpush・部内業務利用へ進む前に、セルフ・セキュリティ・チェックを実行する。

参照先：<https://git.rakuten-it.com/projects/RLIITDATAAI/repos/self_security_check/browse>

| 手順 | 作業 | 補足 |
|---|---|---|
| **1** | 上記リポジトリを開き、READMEまたは実行手順を確認する。 | 実行コマンドはリポジトリ側の記載を正とする。 |
| **2** | チェック対象のローカルリポジトリを確認する。 | 対象は、AI Agentで作成したコード、設定、README、依存関係ファイルを含む作業フォルダ。 |
| **3** | セルフ・セキュリティ・チェックを実行する。 | 実行できない場合やエラー内容が分からない場合は管理者に相談する。 |
| **4** | 指摘事項を確認し、必要に応じて修正する。 | Secret混入、危険な実装、不要ファイル、依存関係の問題を確認する。 |
| **5** | 修正後に再実行する。 | 指摘が解消しない場合は、個人判断で共有・push・業務利用へ進めない。 |
| **6** | 実行したことをREADMEまたはレビュー記録に残す。 | Level 2以上では共有時、Level 3では承認時の確認材料にする。 |

チェックは安全性を完全に保証するものではない。検出できない設計不備や業務上の妥当性は、人間レビューで確認する。

## 5. バージョンアップ

| 手順 | 作業 |
|---|---|
| **1** | 更新案内で、対象者、更新理由、期限を確認する。 |
| **2** | 新しい標準インストーラーを実行する。 |
| **3** | 導入確認コマンドで標準版、設定ハッシュ、更新結果を確認する。 |
| **4** | 動作確認を行う。 |
| **5** | 失敗した場合は、個人判断で設定変更せず管理者へ連絡する。 |

## 6. トラブル時

| 状況 | 対応 |
|---|---|
| **インストールできない** | エラー内容、端末名、実行日時を記録して管理者へ連絡する。 |
| **設定ハッシュが違う** | 標準設定から外れている可能性があるため、再導入または管理者確認を行う。 |
| **外部接続を求められる** | 個人判断で許可せず、利用目的と接続先を確認する。 |
| **高額・長時間処理になりそう** | 実行前に止め、費用影響を確認する。 |
| **Sandboxが有効か分からない** | 作業範囲・実行範囲の確認方法を管理者に確認する。 |
| **セルフ・セキュリティ・チェックが実行できない** | エラー内容と対象リポジトリを記録し管理者へ相談する。 |


## 5. ライブラリ導入確認

Codex / Claude Code等の実行中に、PythonやNode.js等のランタイム、またはpip/npm等のライブラリ導入を提案される場合があります。

| 区分 | 対応 |
|---|---|
| **プログラムランタイム** | 会社のソフトウェア導入許可リスト・指定手順に従う。許可外の場合は導入しない。 |
| **許可済みライブラリ** | 標準インストーラーで配布される許可済みパッケージリストに含まれる場合のみ導入する。 |
| **許可外・不明なライブラリ** | 導入を停止し、用途、パッケージ名、バージョン、理由を添えてDATA AIチームまたは管理者へ相談する。 |
| **ガードレール停止時** | Hook/ラッパーで止まった導入を別コマンドや設定変更で回避しない。 |
| **更新** | 許可済みパッケージリストは標準モジュール配布時に更新される。 |
