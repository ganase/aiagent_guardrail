# 導入手順書（v0.2）

## 目的

AI Agent を標準構成で導入し、利用環境の分散、古い設定の残存、未承認接続を防ぐ。

---

## 1. 導入前確認

| 確認項目 | 内容 |
|---|---|
| **対象端末** | 業務で利用する会社管理 PC であること |
| **利用目的** | 個人試行・チーム内利用・部内業務利用のどれかを確認する |
| **OS / 環境** | Windows 11 を推奨 |
| **権限** | **原則必須: 管理者権限で実行**（ACL 保護と Claude Code managed-settings のシステム配置に必要） |
| **外部接続** | 未承認 MCP、Plugin、個人契約 API、未承認 SaaS を使わない |
| **機密情報** | 機密情報・認証情報・本番データを投入しない前提で始める |
| **プログラムランタイム** | Python、Node.js 等は会社のソフトウェア導入許可リスト・指定手順に従う |
| **ライブラリ導入** | pip / npm 等は許可済みパッケージリストと指定手順に従う |
| **Codex / Claude Code** | 標準インストーラーまたは会社指定手順で導入済みであること |

> **⚠ 非 admin 導入について**
> 非管理者でも技術的にはインストール可能ですが、以下の制約があります。
> - ACL による設定ファイル改変保護が設定されません（ユーザーが guardrail_policy.json 等を書き換え可能な状態になります）
> - Claude Code managed-settings のシステム領域への配置ができません
> - **非 admin 導入は評価・開発環境用途に限定してください。**

---

## 2. インストール（管理者 PowerShell で実行）

```powershell
cd <展開先>
.\installer\install_standard.ps1 -ConfigureClaude -ConfigureCodex
```

ユーザー PATH に `ai-pip` / `ai-npm` を追加する場合:

```powershell
.\installer\install_standard.ps1 -ConfigureClaude -ConfigureCodex -AddWrappersToUserPath
```

インストールが成功すると:
- `{InstallRoot}/config/installed_hashes.csv` が生成されます（改ざん検知用）
- ACL が設定され、Users の書込権限が制限されます
- Claude Code managed-settings に `run_guardrail_hook.cmd` 経由の hook が配置されます

---

## 3. 導入後の確認

```powershell
.\installer\check_status.ps1
```

スモークチェックが実行され、allow / ask / deny / 危険コマンド / blocked_path の各動作が確認できます。

| 確認項目 | 確認内容 |
|---|---|
| **標準版** | 指定された標準版が入っている |
| **設定ハッシュ** | 標準設定が改変されていない（installed_hashes.csv と照合） |
| **外部接続** | 許可されていない接続先が追加されていない |
| **Sandbox** | AI Agent が操作できる作業フォルダ・実行範囲が限定されている |
| **ライブラリ導入制御** | `pip install` / `npm install` 等が Hook 経由で確認される |
| **ログ** | 導入状況・エラー・拒否イベントを確認できる |

---

## 4. ライブラリ導入確認

Codex / Claude Code 等の実行中にライブラリ導入を提案される場合の対応:

| 区分 | 対応 |
|---|---|
| **プログラムランタイム** | 会社のソフトウェア導入許可リスト・指定手順に従う。許可外は導入しない |
| **許可済みライブラリ** | 許可済みパッケージリストに含まれる場合のみ導入する（自動許可） |
| **未審査ライブラリ** | 承認ダイアログが表示される。用途・パッケージ名・バージョン・理由を確認してから判断する |
| **禁止・typosquat 疑い** | ガードレールがブロック（exit 2）。個人判断で回避しない |
| **ガードレール停止時** | Hook で止まった導入を別コマンドや設定変更で回避しない |

```powershell
# 許可済みの例（自動許可 = ALLOW、ダイアログなし）
ai-pip install pandas

# 未審査パッケージ（人間承認 = ASK）
ai-pip install some-new-library

# 禁止パッケージ（ブロック = DENY）
ai-pip install example-malicious-package
```

---

## 5. Claude Code での動作確認

Claude Code が Bash で pip install を提案したとき:
- allowlist 済みパッケージ → 無音で自動承認（ダイアログなし）
- 未審査パッケージ → 人間承認ダイアログが表示される
- deny / typosquat 疑い → ブロック（Claude にエラーメッセージが返る）

## 6. セルフ・セキュリティ・チェック

ローカルリポジトリに仕組みができた場合は、チーム共有・Bitbucket への push・部内業務利用へ進む前に、セルフ・セキュリティ・チェックを実行する。

| 手順 | 作業 | 補足 |
|---|---|---|
| **1** | チェック手順書（組織指定のリポジトリ）を確認する | 実行コマンドはリポジトリ側の記載を正とする |
| **2** | チェック対象のローカルリポジトリを確認する | 対象は AI Agent で作成したコード・設定・README・依存関係ファイルを含む作業フォルダ |
| **3** | セルフ・セキュリティ・チェックを実行する | 実行できない場合やエラー内容が分からない場合は管理者に相談する |
| **4** | 指摘事項を確認し、必要に応じて修正する | Secret 混入・危険な実装・不要ファイル・依存関係の問題を確認する |
| **5** | 修正後に再実行する | 指摘が解消しない場合は、個人判断で共有・push・業務利用へ進めない |
| **6** | 実行したことを README またはレビュー記録に残す | Level 2 以上では共有時、Level 3 では承認時の確認材料にする |

チェックは安全性を完全に保証するものではない。検出できない設計不備や業務上の妥当性は、人間レビューで確認する。

---

## 7. 改ざん検知について

hook 起動時に `config/installed_hashes.csv` の SHA256 と照合します。設定ファイルが改ざんされた場合、claude-hook モードで exit 2（ブロック）になります。定期的に `check_status.ps1` で整合性を確認してください。

---

## 8. バージョンアップ

| 手順 | 作業 |
|---|---|
| **1** | 更新案内で、対象者・更新理由・期限を確認する |
| **2** | 新しい標準インストーラーを実行する |
| **3** | `check_status.ps1` で標準版・設定ハッシュ・更新結果を確認する |
| **4** | 動作確認を行う |
| **5** | 失敗した場合は、個人判断で設定変更せず管理者へ連絡する |

---

## 9. トラブル時

| 状況 | 対応 |
|---|---|
| **インストールできない** | エラー内容・端末名・実行日時を記録して管理者へ連絡する |
| **設定ハッシュが違う（改ざん検知）** | 標準設定から外れている可能性があるため、再導入または管理者確認を行う |
| **外部接続を求められる** | 個人判断で許可せず、利用目的と接続先を確認する |
| **高額・長時間処理になりそう** | 実行前に止め、費用影響を確認する |
| **Sandbox が有効か分からない** | 作業範囲・実行範囲の確認方法を管理者に確認する |
| **セルフ・セキュリティ・チェックが実行できない** | エラー内容と対象リポジトリを記録し管理者へ相談する |
| **ガードレールが意図せずブロックする** | コマンドと出力を記録し、管理者または AIガバナンスチームに相談する。自己判断で hook を無効化しない |
