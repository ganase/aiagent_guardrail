# サンプル

## 目的

AI Agentに小さなローカルアプリを作らせる練習用サンプルです。業務システムではなく、Level 1の個人・ローカル試行を想定します。

## サンプル：勤怠メモアプリ

```text
アイコンを起動するとブラウザが開き、出勤ボタンを押すと勤怠.csvに、Windowsのユーザー名、「出勤」という文字と、その時のタイムスタンプを書き込むアプリを作成してください。
退勤ボタンを押すと、同じCSVにWindowsのユーザー名、「退勤」という文字と、その時のタイムスタンプを書き込んでください。

条件：
- PCローカルで動作すること。
- 外部通信を行わないこと。
- 本番データや機密情報を扱わないこと。
- 作業フォルダ外のファイルを変更しないこと。
- README.md と .gitignore を作成すること。
- 作成後に、どのファイルを作成・変更したか説明すること。
```

## 期待する成果物

| ファイル | 内容 |
|---|---|
| `app.py` | アプリ本体。 |
| `start.bat` | 起動用バッチ。 |
| `勤怠.csv` | 出勤・退勤の記録先。初回起動時に作成。 |
| `README.md` | 目的、起動方法、制限、注意点。 |
| `.gitignore` | ログ、一時ファイル、ローカル設定などを除外。 |

## 追加で依頼する確認

```text
作成したコードについて、以下を確認してください。
1. 外部通信がないこと。
2. 作業フォルダ外のファイルを変更しないこと。
3. APIキー、秘密鍵、個人情報、本番データが含まれていないこと。
4. 不要なログ、一時ファイル、キャッシュがGit管理対象になっていないこと。
5. README.mdに起動方法と制限事項が書かれていること。
```

## Bitbucketへ保存する場合

```text
README.md と .gitignore を確認したうえで、Bitbucketの指定リポジトリへコミット・プッシュしてください。
コミット前に、変更ファイル一覧と変更内容を要約してください。
コミットメッセージは「勤怠メモアプリの初期版を追加」としてください。
```

## 注意

このサンプルは練習用です。実際の勤怠管理や人事データ管理には使わないでください。
