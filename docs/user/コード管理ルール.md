# コード管理ルール

## 目的

AI Agentで作成したコードを、ローカルPCだけに残さず、引き継ぎ・レビュー・復旧できる状態にする。

## 1. 基本ルール

| 項目 | ルール |
|---|---|
| **保存先** | 継続利用・共有・業務利用するコードはBitbucketに保存する。 |
| **部内業務利用** | Level 3へ進む場合、Bitbucket保存を承認要件とする。 |
| **履歴管理** | commit / push により変更履歴を残す。 |
| **説明責任** | READMEに目的、起動方法、前提、制限、注意点を書く。 |
| **機密情報** | APIキー、秘密鍵、接続文字列、個人情報、本番データを保存しない。 |

## 2. 保存するもの・しないもの

| 区分 | 例 |
|---|---|
| **保存する** | ソースコード、README、設定テンプレート、サンプルデータ、起動手順、テストコード |
| **保存しない** | `.env`、秘密鍵、APIキー、認証情報、本番データ、個人情報、ログ、一時ファイル、キャッシュ、ローカル固有設定 |

## 3. .gitignore

`.gitignore` は、Git管理に含めないファイルを指定するためのファイルです。Secret混入、不要ファイル混入、ローカル依存ファイル混入を防ぐために使います。

| 除外対象 | 例 |
|---|---|
| **認証情報** | `.env`, `*.key`, `*.pem`, `credentials.json` |
| **ログ・一時ファイル** | `*.log`, `tmp/`, `cache/` |
| **ローカル設定** | `.vscode/`, `local_config.*` |
| **生成物** | `__pycache__/`, `dist/`, `build/` |
| **本番データ** | `prod_data/`, `customer_data/` |

## 4. READMEに書くこと

| 項目 | 内容 |
|---|---|
| **目的** | 何のためのコードか。 |
| **起動方法** | どのコマンド・手順で動くか。 |
| **入力・出力** | 何を読み、何を作るか。 |
| **前提** | 必要な環境、ライブラリ、権限。 |
| **制限** | 使ってよい範囲、使ってはいけない範囲。 |
| **注意点** | 機密情報、本番データ、外部接続、費用影響。 |

## 5. AI Agentへの指示例

```text
作成したコードを確認し、README.md と .gitignore を整備してください。
認証情報、秘密鍵、個人情報、本番データ、不要なログや一時ファイルが含まれていないか確認してください。
問題なければ、Bitbucketの指定リポジトリへコミット・プッシュしてください。
コミット前に変更差分を要約してください。
```

## 6. 業務利用前チェック

| 確認項目 | 内容 |
|---|---|
| **Bitbucket保存** | 指定リポジトリにpush済み。 |
| **README** | 目的、起動方法、制限が書かれている。 |
| **.gitignore** | Secret、ログ、一時ファイルが除外されている。 |
| **Security Check** | Secret混入、危険処理、依存関係を確認済み。 |
| **レビュー** | 作成者以外が処理内容を確認済み。 |

## セルフ・セキュリティ・チェック

ローカルリポジトリに仕組みができた場合は、チーム共有・Bitbucketへのpush・部内業務利用へ進む前に、セルフ・セキュリティ・チェックを実行する。

参照先・詳細な位置づけは [運用設計書.md](../admin/運用設計書.md) を参照してください（実行手順は自社が指定するセルフ・セキュリティ・チェックツールのリポジトリを正とします）。

| 確認 | 内容 |
|---|---|
| **実行タイミング** | ローカルリポジトリにコード・設定・READMEがそろった時点。 |
| **確認対象** | Secret混入、不要ファイル、危険な実装、依存関係の問題。 |
| **結果の扱い** | 指摘事項を修正し、解消しない場合は管理者に相談する。 |
| **記録** | 実行したことをREADMEまたはレビュー記録に残す。 |


## 7. 依存ライブラリ管理

AI Agentが作成したコードでpip/npm等のライブラリを利用する場合は、許可済みパッケージリストに従う。

| 項目 | ルール |
|---|---|
| **導入可否** | 許可済みパッケージリストに含まれるライブラリのみ導入する。 |
| **バージョン** | 可能な限りバージョン範囲または固定バージョンを明記する。 |
| **依存関係ファイル** | Pythonは `requirements.txt` 等、Node.jsは `package.json` / `package-lock.json` 等を保存する。 |
| **許可外ライブラリ** | 導入せず、用途・パッケージ名・バージョン・理由を整理してAIガバナンスチームまたは管理者へ相談する。 |
| **チェック** | セルフ・セキュリティ・チェックを実行する。依存関係チェック（`pip-audit` / `npm audit` 等の既知脆弱性スキャン）は本リポジトリでは未実装（[必要な仕組み一覧.md](../admin/必要な仕組み一覧.md) の「次に必要な仕組み」参照）。整備されるまでは、許可済みパッケージリストのreviewed_at/expires_atとセルフ・セキュリティ・チェックで代替する。 |
