# 既知の限界（v0.2 時点）

本ガードレールが対処できない・対処が困難な既知の穴を正直に記します。対策なしに「安全」と思わないようにしてください。

## 1. Codex 側には強制力のある enforcement が存在しない

AGENTS.md はモデルへの助言であり、強制力はありません。`ai-pip` / `ai-npm` は任意利用であり、Codex が素の `pip install` / `npm install` を直接実行した場合、hook は発動しません。

**恒久対策**: ネットワーク境界（社内 PyPI/npm ミラー + egress プロキシ）による allowlist 強制。

## 2. 文字列マッチの原理的回避可能性

hook は Bash コマンドの文字列に対するパターンマッチです。以下のような手法で容易に回避できます。

```python
# hook は発動しない
python -c "__import__('pip').main(['install','evil-pkg'])"

# スクリプトファイル経由
python install_something.py

# 環境変数経由
PACKAGE=evil-pkg pip install $PACKAGE

# base64 エンコード
pwsh -enc <base64 of pip install evil-pkg>
```

**恒久対策**: ネットワーク境界が唯一の呼び出し方法に依存しない防御。

## 3. 推移的依存の未検査

`pip install pandas` を許可した場合、pandas が依存している numpy、python-dateutil 等は個別にチェックされません。悪意ある依存関係が混入しても、直接指定でなければ検知できません。

**恒久対策**: `pip-audit` / `npm audit` による定期スキャン。lock ファイルの管理。

## 4. 非 admin 導入時の改変可能性

管理者権限なしでインストールした場合:
- ACL 保護が設定されません
- ユーザーが `guardrail_policy.json`、`package_allowlist.json`、`aiagent_guardrail_check.py` を自由に書き換え可能です
- `AIAGENT_GUARDRAIL_HOME` 環境変数（CLI モード）を差し替えるだけで CLI ガードを無効化できます

**恒久対策**: 管理者による正式導入（ACL 保護付き）を徹底する。非 admin 導入は評価用途限定と明記する。

## 5. 承認疲れ（ask の乱発）

未知パッケージが多い環境では `ask` ダイアログが頻出し、人間が中身を確認せずに承認するリスクがあります。

**軽減策**: `logs/package_decisions.csv` の定期レビューで ask の多いパッケージを allowlist に取り込む。allowlist の整備で ask を減らす。

## 6. ユーザー自身のターミナルは対象外

Claude Code が提案した Bash コマンドは hook でチェックされますが、ユーザーがターミナルで直接 `pip install X` を実行しても hook は発動しません。

**恒久対策**: ネットワーク境界。端末管理ポリシー。

## 7. installed_hashes.csv 自体の改ざん

改ざん検知は `installed_hashes.csv` の内容を正として照合します。ACL 保護がない環境（非 admin 導入時）では `installed_hashes.csv` 自体を書き換えることで改ざん検知を無効化できます。

**恒久対策**: ACL 保護付きの管理者導入。

## 8. バージョン範囲の実行時未検証

`allowed_versions` フィールドはドキュメントとして記録されますが、実行時に「インストールしようとしているバージョンが範囲内か」の検証は行いません。`pandas==1.0.0`（古い脆弱バージョン）も `pandas==2.0.0` も同じく allow になります。

**恒久対策**: `pip-audit` による定期スキャン。

## 9. ハッシュファイル非存在時の動作

`installed_hashes.csv` が存在しない場合（開発環境・インストール直後等）は警告なしに通過します。改ざんとハッシュファイル不在の区別ができません。

## まとめ: ガードレールの位置づけ

本ガードレールは「完全なセキュリティ」ではなく「善意の AI エージェントに対する摩擦と監査ログの提供」と位置づけてください。悪意ある内部者や高度な攻撃者に対しては有効ではありません。ネットワーク境界・端末管理・教育・定期レビューとの組み合わせが必要です。
