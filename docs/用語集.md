# 用語集

| 用語 | 説明 |
|---|---|
| **AI Agent** | 利用者の指示をもとに、調査、設計、コード作成、ファイル操作、コマンド実行などを行うAI。 |
| **Coding Agent** | コード作成や修正を主目的とするAI Agent。 |
| **活用度** | AI Agentがどれだけ業務に使われているかを示す考え方。利用人数、対象業務数、効果で見る。 |
| **Level 1 個人・ローカル試行** | 個人PC上で小さく試す段階。外部送信、本番影響、機密情報利用はしない。 |
| **Level 2 チーム内利用** | チーム内で便利ツールや使い方を共有する段階。 |
| **Level 3 部内業務利用** | 業務に組み込む段階。責任者レビュー、Bitbucket保存、運用方法の明確化が必要。 |
| **ガードレール** | 危険な操作を人間の注意だけでなく、仕組みで止める安全柵。 |
| **Sandbox** | AI Agentが操作できるファイル範囲・実行範囲を限定する仕組み。 |
| **Permission** | AI Agentに許可する操作や権限。 |
| **Approval** | 高リスク操作の前に人間の承認を求める仕組み。 |
| **Hook** | AI Agentの実行前後にチェック処理を差し込む仕組み。 |
| **deny / ignore** | AI Agentに読ませないファイル・フォルダを指定する仕組み。 |
| **MCP** | AI Agentが外部ツールやデータソースに接続するための仕組み。未承認接続に注意する。 |
| **Plugin / API** | AI Agentが外部サービスや機能を使うための接続方法。 |
| **プロンプトインジェクション** | Web、PDF、メール、README、ログなどに埋め込まれた悪意ある指示を、AI Agentが命令として扱ってしまうリスク。 |
| **Secret** | APIキー、秘密鍵、認証情報、接続文字列など、漏洩してはいけない情報。 |
| **.gitignore** | Git管理に含めないファイルを指定するファイル。Secret、ログ、一時ファイル、ローカル設定を除外する。 |
| **Bitbucket** | コードを保存し、履歴管理・レビュー・引き継ぎに使う社内コード管理基盤。 |
| **commit** | 変更内容を履歴として記録すること。 |
| **push** | ローカルの変更履歴をBitbucketへ反映すること。 |
| **README** | 目的、起動方法、前提、制限、注意点を説明する文書。 |
| **Security Check** | Secret混入、危険処理、依存関係、権限などを確認する作業。 |
| **ハートビート** | 端末やAgentから導入状態・バージョン情報を定期的に確認する仕組み。 |
| **設定ハッシュ** | 標準設定が改変されていないか確認するための値。 |
| **サプライチェーンリスク** | ライブラリ、MCP、Plugin、サンプルコードなど外部部品に脆弱性や悪意ある処理が含まれるリスク。 |
| **承認疲れ** | Approvalが多すぎたり分かりにくかったりして、内容を読まず承認してしまう状態。 |
| **無限ループ・大量実行** | AI Agentが再試行、探索、API呼び出しを繰り返し、費用や負荷が膨らむ状態。 |


## 追加用語：ランタイム・ライブラリ制御

| 用語 | 説明 |
|---|---|
| **プログラムランタイム** | Python、Node.jsなど、プログラムを実行するための基盤ソフトウェア。会社のソフトウェア導入許可リストに従う。 |
| **許可済みパッケージリスト** | pip/npm等で導入してよいライブラリ名・バージョン・状態をまとめたリスト。標準インストーラーで配布する。 |
| **インストールHook / ラッパー** | `pip install` / `npm install` 等の実行前に許可済みパッケージリストと照合し、許可外導入を止める仕組み。 |
| **allow / review / deny** | ライブラリの状態区分。allowは導入可、reviewは要確認、denyは導入不可。 |
| **依存関係チェック** | ライブラリとその依存パッケージに既知脆弱性や不審な点がないか確認すること。 |
