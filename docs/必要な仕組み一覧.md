# 必要な仕組み一覧

## 目的

AI Agent を安全に展開するために、文書ルールだけでなく、標準インストーラー、許可リスト、ガードレール、チェックツール、更新運用を仕組みとして整備する。

## 1. 必須に近い仕組み

| 仕組み | 目的 | 主な内容 | 優先度 |
|---|---|---|---|
| **標準インストーラー** | Codex / Claude Code 等を標準構成で導入する | Agent 本体、標準設定、AGENTS.md / CLAUDE.md、Hook、テンプレート、許可済みパッケージリストを配布 | 高 |
| **標準設定・ガードレールファイル** | 利用者ごとの設定差異を防ぐ | Sandbox、Permission / Approval、deny / ignore、接続許可リスト、コマンド制限、ログ設定 | 高 |
| **ランタイム許可リスト連携** | 会社許可外の Python / Node.js 等を導入しない | 会社のソフトウェア導入許可リストを参照し、標準手順へ誘導 | 高 |
| **許可済みパッケージリスト（3層）** | pip/npm 等のライブラリ導入を制御する | allow（自動）/ review・未知（人間承認）/ deny（ブロック）+ typosquat 検出 | 高 |
| **インストール Hook / ラッパー（fail-closed）** | 許可外ライブラリ導入を実行前に止める | `pip install` / `npm install` 等を捕捉し、allowlist 照合。hook 故障時も exit 2 で fail-closed | 高 |
| **セルフ・セキュリティ・チェック** | ローカルリポジトリの危険を利用者自身で確認する | Secret 混入、不要ファイル、危険実装、依存関係、README、.gitignore 確認 | 高 |
| **設定ハッシュ・バージョン確認** | 標準設定や許可リストの改変・古い版を検知する | 設定ファイル、Hook、許可リスト、テンプレートのハッシュ・版数確認 | 高 |

## 2. 次に必要な仕組み

| 仕組み | 目的 | 主な内容 | 優先度 |
|---|---|---|---|
| **社内 PyPI/npm ミラー + egress プロキシ（ネットワーク境界）** | hook/ラッパーを回避した直接 install も含め、allowlist 外のパッケージを根本的に防ぐ | 社内ミラーに allowlist パッケージのみを掲載。外部 PyPI/npm へのネットワークアクセスを egress プロキシで遮断 | **高（最優先ロードマップ）** |
| **許可リスト更新ワークフロー** | ライブラリの追加・更新・禁止を管理する | 申請、リスク判定、承認、配布、旧版無効化 | 高 |
| **例外申請フォーム** | 許可外ライブラリや外部接続の判断を個人判断にしない | 用途、対象パッケージ、バージョン、理由、代替手段、利用範囲を記録 | 中 |
| **依存関係チェック** | 直接指定ライブラリだけでなく依存パッケージも確認する | pip-audit 相当、npm audit 相当、既知脆弱性 DB 照合 | 中 |
| **導入ログ・ハートビート** | 導入済み端末、設定版数、未更新者を把握する | Agent 版数、設定版数、許可リスト版数、最終確認日時 | 中 |
| **拒否イベント・エラーログ** | ガードレールで止めた操作を改善につなげる | 許可外 install、危険コマンド、外部接続、Secret 検知の記録 | 中 |
| **README / .gitignore テンプレート** | ブラックボックス化と Secret 混入を防ぐ | README 雛形、.gitignore 雛形、依存関係ファイル雛形 | 中 |
| **教育・FAQ** | 利用者が迷う場面を減らす | 承認操作、プロンプトインジェクション、外部情報、ライブラリ導入の FAQ | 中 |

> **ネットワーク境界が最優先な理由**
>
> hook/ラッパーは「どのツールから呼ばれるか」に依存します。`python -c "import pip; pip.main(['install','X'])"` や任意のスクリプト経由では hook は発動しません。Codex 側にも強制力のある enforcement は存在しません（AGENTS.md は助言、ai-pip/ai-npm は任意利用）。
>
> **ネットワーク境界（社内 PyPI/npm ミラー + egress プロキシ）のみが、呼び出し方法に依存しない一次防御になります。** hook/ラッパーは二次防御として有効ですが、一次防御にはなれません。

## 3. あると望ましい仕組み

| 仕組み | 目的 | 主な内容 | 優先度 |
|---|---|---|---|
| **ライブラリリスク判定バッチ** | 許可済みパッケージリスト作成を半自動化する | PyPI/npm メタデータ、既知脆弱性、更新日、依存数、類似名を収集してスコア化 | 中 |
| **承認画面の要約** | 承認疲れ・誤承認を減らす | 対象ファイル、外部送信先、コマンド、費用影響、install 対象を要約表示 | 中 |
| **権限棚卸しレポート** | MCP / Plugin / API / Bitbucket 権限を見直す | 接続先、権限、利用者、最終利用日を棚卸し | 低〜中 |
| **ロールバック手順** | 新版配布で問題が出た場合に戻す | 前版パッケージ、旧設定、戻し手順、旧許可リスト | 中 |

## 4. 許可済みパッケージリストの最小項目

| 項目 | 例 | 説明 |
|---|---|---|
| ecosystem | python / npm | 対象エコシステム |
| package_name | pandas | パッケージ名 |
| allowed_version | >=2.0,<3.0 | 許可するバージョン範囲（現状は参照のみ、実行時検証は未実装） |
| status | allow / review / deny | 導入可否 |
| reason | 社内標準ライブラリ | 許可理由 |
| reviewed_date | 2026-06-10 | 確認日 |
| owner | AIガバナンスチーム | 管理者 |
| notes | 依存関係確認済み | 補足 |

## 5. ライブラリ導入ガードレールの最小フロー（v0.2 版）

```text
Codex / Claude Code
  ↓
pip install / npm install を提案
  ↓
Hook / ラッパーが実行前に捕捉
  ↓
ローカル許可済みパッケージリストを参照
  ↓
deny リスト / typosquat  → exit 2 ブロック
allow リスト             → exit 0 + JSON allow（無音で通す）
review / 未知            → exit 0 + JSON ask（人間承認ダイアログ）
  ↓
全判定を logs/package_decisions.csv に記録
  ↓
ガバナンスチームが定期レビュー → allowlist 反映
```

## 6. セルフ・セキュリティ・チェックの見直し観点

| 観点 | 見直し内容 |
|---|---|
| **Secret 検知** | `.env`、秘密鍵、API キー、接続文字列、認証ファイルを検知できるか。 |
| **不要ファイル検知** | ログ、一時ファイル、キャッシュ、本番データ、個人情報を検知できるか。 |
| **依存関係確認** | `requirements.txt`、`package.json`、lock ファイルを確認できるか。 |
| **許可リスト照合** | 依存ライブラリが許可済みパッケージリストに含まれるか確認できるか。 |
| **危険実装確認** | 外部送信、危険コマンド、認証回避、過剰権限処理の兆候を確認できるか。 |
| **README 確認** | 目的、起動方法、入力・出力、制限、注意点が記載されているか。 |
| **結果出力** | 指摘事項、重大度、修正案を利用者に分かる形で出せるか。 |

## 7. まず作る順番

| 順番 | 作るもの | 理由 |
|---|---|---|
| 1 | 標準インストーラー | すべてのガードレール配布の土台になるため |
| 2 | 標準設定・ガードレールファイル | Sandbox、Approval、接続制限などの基本制御を入れるため |
| 3 | 許可済みパッケージリスト（3層） | ライブラリ導入制御の判断軸になるため |
| 4 | インストール Hook / ラッパー（fail-closed） | 許可外ライブラリを実行前に止めるため |
| 5 | セルフ・セキュリティ・チェック改訂 | コード・依存関係・Secret を利用者が確認できるようにするため |
| 6 | バージョンアップ・配布管理 | 設定や許可リストを更新し続けるため |
| 7 | ネットワーク境界（PyPI/npm ミラー + プロキシ）| hook に依存しない一次防御。最優先ロードマップ |
| 8 | 例外申請・FAQ | 止まったときに現場が迷わないようにするため |
