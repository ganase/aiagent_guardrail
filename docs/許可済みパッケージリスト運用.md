# 許可済みパッケージリスト運用

## 方針

- default policy は deny とする。
- allow / review / deny の3区分で管理する。
- 許可リスト外のパッケージは導入しない。
- review扱いはAIガバナンスチームまたは管理者確認後に利用する。

## 更新フロー

| 手順 | 内容 |
|---|---|
| 1 | 利用者が用途、パッケージ名、バージョン、理由を申請する |
| 2 | AIガバナンスチームがリスク確認する |
| 3 | allow / review / deny を決定する |
| 4 | `package_allowlist.json` を更新する |
| 5 | `validate_allowlist.py` で検証する |
| 6 | 標準インストーラーまたは標準モジュール更新で配布する |

## リスク確認観点

| 観点 | 内容 |
|---|---|
| 既知脆弱性 | OSV、pip-audit、npm audit等で確認 |
| メンテナンス状況 | 更新日、リリース頻度、メンテナ数 |
| タイポスクワッティング | 有名パッケージ名との類似 |
| 外部通信 | HTTP通信・認証・ブラウザ操作の有無 |
| 依存関係 | 依存パッケージ数、依存先の安全性 |
| ライセンス | 社内利用可能なライセンスか |
