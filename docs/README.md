## 今回の改訂方針

| 方針 | 内容 |
|---|---|
| ファイル名整理 | 先頭の `AIAgent` を外し、サンプルは `サンプル.md` に変更。 |
| 子文書の再整理 | 最新の運用設計に合わせて、リスク、技術対策、運用対策と矛盾しないように改訂。 |
| 利用ルールブック | ユーザーが読める量に絞り、活用度モデルを中心に構成。 |
| 重複削減 | 詳細は運用設計、実務ルールは子文書に分離。 |
