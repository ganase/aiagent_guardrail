# ドキュメント一覧

文書は **ユーザー向け**（AIを使う方）と **管理者向け**（環境を整備・管理する方）に分かれています。

---

## ユーザー向け（`user/`）

AIを使う方が読む文書です。まず**利用ルールブック**を確認してください。

| 文書 | 内容 |
|---|---|
| [利用ルールブック](user/利用ルールブック.md) | 守るべきルールと活用度レベルの判断基準 |
| [導入手順書](user/導入手順書.md) | SETUP.bat からの導入手順・トラブル対応 |
| [コード管理ルール](user/コード管理ルール.md) | Bitbucket 保存・.gitignore・README の書き方 |
| [サンプル](user/サンプル.md) | AI Agent への指示例・プロンプトサンプル |
| [用語集](user/用語集.md) | 本文書セットで使う用語の定義 |

---

## 管理者向け（`admin/`）

環境構築・ガードレール設計・ポリシー管理を担当する方が読む文書です。

| 文書 | 内容 |
|---|---|
| [運用設計書](admin/運用設計書.md) | リスク（R1–R15）・技術設計・運用設計のマトリクス |
| [ガードレール設計](admin/ガードレール設計.md) | 3層ポリシー・Hook・fail-closed 設計の詳細 |
| [許可済みパッケージリスト運用](admin/許可済みパッケージリスト運用.md) | allowlist の更新フロー・ログレビュー手順 |
| [既知の限界](admin/既知の限界.md) | 実証済みの穴と恒久対策ロードマップ |
| [必要な仕組み一覧](admin/必要な仕組み一覧.md) | 未実装項目・優先度・ロードマップ |
| [バージョンアップ設計](admin/バージョンアップ設計.md) | 配布・更新・ロールバック手順 |
