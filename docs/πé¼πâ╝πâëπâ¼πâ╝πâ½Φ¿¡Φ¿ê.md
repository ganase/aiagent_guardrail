# ガードレール設計

## 基本方針

標準インストーラーにより、標準設定、ガードレール、テンプレート、許可済みパッケージリスト、バージョン管理を配布します。

| ガードレール | 実装 | 主な対象リスク | 限界 |
|---|---|---|---|
| Sandbox | Codex Windows sandbox / Claude権限制御 / 作業範囲制限 | ローカル破損、本番影響、危険コマンド | 外部送信や誤承認は単独では防げない |
| Permission / Approval | Claude permissions / Codex approval | 危険操作、外部送信、本番影響 | 承認疲れ・誤承認は残る |
| Hook | `aiagent_guardrail_check.py` | pip/npm、危険コマンド、ランタイム導入 | Hookが無効な環境では弱い |
| 許可済みパッケージリスト | `package_allowlist.json` | サプライチェーン、未許可ライブラリ | 許可済みでも将来脆弱性が出る可能性 |
| deny / ignore | Claude permissions / AGENTS.md | Secret読取、機密ファイル読取 | ユーザーが別経路で渡すと防げない |
| 設定ハッシュ | `installed_hashes.csv` | 設定改変、バージョン差異 | 定期点検が必要 |

## Claude Code

Claude Codeでは managed settings と PreToolUse hook を使う想定です。

## Codex

Codexでは Windows sandbox、`config.toml`、`requirements.toml`、`AGENTS.md`、`ai-pip` / `ai-npm` を併用します。CodexネイティブHook連携は環境仕様に合わせて追加確認してください。
